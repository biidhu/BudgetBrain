# Repository Structure

The repository was organized to separate documentation, frontend, backend, and database components.

- **docs/** – All project documentation, sprint logs, team agreements, and system design files.  
- **frontend/** – Handles everything the user sees: dashboard, charts, forms, and interactions.  
- **backend/** – Provides APIs to store and retrieve data, handles business logic, and communicates with the database.  
- **database/** – Contains SQL scripts for creating tables, relations, and initial seed data.  
- **package.json** – Manages Node.js dependencies, scripts for running server and build processes.
